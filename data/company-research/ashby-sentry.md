# Sentry — research
Provider/key: ashby:sentry | company_type: product

What they do: Sentry is an application monitoring and error tracking platform used by developers to detect, triage, and resolve software bugs and performance issues in production. They provide SDKs for 100+ platforms, a cloud-based error aggregation service, and increasingly AI-powered developer tooling. Used by 4M+ developers, 200K+ organizations, 790B+ monthly events.

How they describe themselves: "An open-source company" making "every developer's lives just a little bit better." Mission is to help developers fix code faster. Values: transparency, developer empowerment, community. Describes itself as "relatively small and moderately fast-growing team." $217M raised.

Size / stage / funding: ~300-500 employees (relatively small per own description). Raised $217M total. Late-stage private. Founded 2012, San Francisco. 4M+ users, 200K+ customers.

Locations / HQ: HQ in San Francisco. Offices in San Francisco, Toronto, Vienna (Austria), Sydney (Australia), Amsterdam, New York. Multi-hub model.

Remote policy: Hybrid model with in-office anchor days. Policy: Mondays, Tuesdays, and Thursdays are in-office anchor days. Most employees work hybrid: 3 days in office (Tue-Wed-Thu), 2 days remote (Mon-Fri). Some fully remote roles exist for employees in locations without a nearby office, but these appear to be exceptions.

Remote-Canada eligibility: No / unclear for most roles. The Toronto office operates on the same hybrid 3-days-in-office model. Roles like "Senior Software Engineer, Control Plane" list San Francisco as base with no remote/Canada option. "Software Engineer, Ingest" is based in Vienna. Roles posted to Ashby board list specific office hubs (SF, Toronto, Vienna). A Sentry iOS SDK role appears in some remote listings but this appears atypical. The hybrid 3-day anchor policy is a disqualifier for remote-Canada from a location like Kimberley BC (no Sentry office nearby).

Engineering & tech: Python (primary backend), Rust (some systems work), Go (some services). Distributed ingestion infrastructure (Points of Presence), relay ingestion layer, Kafka, ClickHouse, PostgreSQL. SDKs in C/C++, JavaScript, iOS, Android. Platform is Kubernetes-based cloud. Engineering teams: AI/ML, Control Plane, Ingest, SDK, Issue Workflow.

Notable / other: Open-source roots (Sentry OSS is self-hostable). Active engineering blog. Acquired Grit (April 2025). Innovation Week 2026 event focused on AI observability. Strong developer community. Control Plane team owns the meta-infrastructure coordinating Sentry's own microservices.

Open relevant roles (sample): Staff Machine Learning Engineer AI, Senior Software Engineer (C/C++) SDK, Senior Software Engineer AI, Senior Software Engineer AI Evals, Software Engineer Ingest, Staff Software Engineer Issue Workflow, Senior Software Engineer Control Plane, Senior Software Engineer (Frameworks) JavaScript SDK
